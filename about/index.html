I'm a student and maker who loves turning ideas into working software. My approach — what I call vibe coding — starts with a feeling: what should this app feel like to use? From there, I iterate quickly, test with real scenarios, and polish until the interface feels calm and confident.

This portfolio showcases three projects built that way: a task tracker for daily focus, a race finder for runners, and a buzz tracker for trending topics. Each one started as a rough sketch and grew through conversation with AI tools, design systems, and plenty of trial and error.

When I'm not coding, I'm studying interaction design, exploring typography, and collecting references that inspire the next build.

How I work

My Approach

Three principles that shape every project in this portfolio.

Start with the vibe

Before writing code, I define the emotional tone — calm, energetic, playful, or precise. That feeling becomes the north star for layout, color, and motion.

Ship, then refine

Working prototypes beat perfect plans. I get something running fast, learn from using it, and improve in focused passes rather than endless planning.

Design with care

Accessible typography, generous whitespace, and intentional motion aren't extras — they're how I show respect for the people using what I build.

Learn in public

Each project here documents a step in my learning journey. The code isn't always perfect, but the process is honest and always moving forward.
