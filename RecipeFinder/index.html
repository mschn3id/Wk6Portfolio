Recipe

Recipe Finder is a web app for discovering recipes, filtering by difficulty and prep time, and saving favorites to a personal collection.

Project details

At a glance

  • Role Product designer
  • Timeline 2026, 2 weeks
  • Tools HTML, CSS, JavaScript, Local Storage
  • Team Solo product designer

Context

Overview

Recipe Finder helps home cooks browse a curated recipe library, narrow results with search and filters, and save dishes they want to make again. The interface supports both grid and list layouts so users can scan visually or compare recipes by time and ingredient count at a glance.

Challenge

The challenge

Recipe sites often bury useful details behind ad cluttered pages or force users to bounce between tabs, bookmarks, and notes. I needed a single place to discover recipes, filter by practical constraints like difficulty and total time, and return to saved favorites without losing context.

Response

The solution

Recipe Finder brings discovery, filtering, and saving into one calm interface. Users can search or filter recipes, switch between grid and list views, and open saved dishes in a focused detail modal with ingredients, instructions, and categories.

How it was built

Process

  • Mapped out key user flows for search, filter, save, and detail views
  • Gathered inspiration from the classic Craigslist and CSS animation sites to inform the design library
  • Prompted site structure and DESIGN.md using Cursor
  • Iterated appropriately, published, and hosted on Netlify

Vibe coding

Working with AI

While using AI to build the Recipe Finder app was highly beneficial, the most enlightening part of the process was creating the DESIGN.md file. I used a more advanced AI model to generate the file. It was fascinating to see how the model took inspiration and successfully produced a structured library, complete with clear, site-like component guidelines.

User journey

Highlights

Key features

  • Search recipes by name or keyword
  • Filter by difficulty, max total time, and max ingredients
  • Grid and list browsing layouts
  • Save recipes to a favorites collection
  • Recipe detail modal with ingredients, instructions, and categories
  • User can leave reviews after completing a recipe

Results

Outcomes

The result is a polished recipe browser that makes everyday meal planning feel straightforward — find, filter, save, and cook without leaving the app.

Looking ahead

Reflection

Next I'd connect a live recipe API for dynamic search, add the ability to add recipes from other collections (i.e. recipe books, friends, etc.) and add user accounts for cross-device sync.
